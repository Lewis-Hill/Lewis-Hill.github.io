Inventory System (UE4 blueprints)

This is an Inventory System made entirely with Blueprints. Features include stacking items, transferring items between Inventories using a click or by dragging and dropping, and a quest system built in to NPC Inventories.

Functionality

Adding items to inventory

After picking up an item using the Interact Interface, the function AddToInventory is called on the Player's Inventory Component. The Data Structure attached to the Item tells the function all the attributes the slot should have. If the Item already exists in the Player's Inventory, it checks to see if the picked up item can be added to an existing slot. If not, a new slot is created for that item.

Toggling the Inventory

The ToggleInventory function deals with any situation where the Inventory is opened or closed, and creates or removes the Inventory UI as appropriate. This covers situations where a chest is interacted with and both Player and Chest inventories are opened, pressing Tab to open the Player Inventory, or pressingthe "X" button to close any Inventory. There is a bool called "IsTransferInventory" to tell the function if there is another inventory to close other than the Player's.

Drag and Drop items

Dragging an item using the Mouse

When the mouse is held on a slot, that slot creates a DragDrop operation and the Drag UI icon which takes the attributes of the item. The original slot is wiped to make the dragging more convincing. The DragDrop operation also takes the Structure attributes from the specific slot as an Input variable, so that it knows how to transfer the item when it is dropped on

NPCs and Chests

Easy to create

I made creating new objects such as Chests and NPCs, with their own Inventory, as simple as creating a child class of the "Container" Blueprint, which comes with its own Inventory Component. From there, you just give it a new static mesh, choose the name of the Container and decide how many slots the Inventory will have.

Once set in the level, you can easily add the items that will start off in the Containers. You can also choose whether it will have a Quest for the Player (For NPCs), and set the Item the Player will need to donate for the Quest, and how many.

Quest System

Accepting quests

Upon clicking the "Accept Quest" button on an NPC that has a Quest, all other Quests are put on hold and the Quest becomes active. The NPC asks the Player for an Item that can be placed in their Inventory. The Quest Item is a Structure variable that can be set in the Editor.

Updating status and delivering rewards

Depending on the item placed in the NPC's Inventory, the status of the Quest is updated in that NPC's Inventory and on the Player's HUD, and the Quest Status Text (How many of the items have been donated) will be updated appropriately for which Item was given.

When the Quest requirements have been met, the Player can click the "Hand In Quest" button to complete the Quest and receive a new Inventory Slot as a reward. The Quest Status Text is again updated and the NPC will no longer offer a Quest.
